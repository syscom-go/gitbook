# 服務管理

## 事故管理

設備異常或IT維運發生緊急事故時，可開立此單進行事故排除流程，以下為大概的流程介紹：

| 開單 | → | 指派關卡 | → | 處理關卡 | → | 關單 |
| :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| 通知事故所有組 |  | 重新指派 |  | 排除事故 |  | 通知開單人員 |
| 通知受派組織 |  | 立即解決 |  | 返回指派 |  |  |
|  |  | 取消事故 |  |  |  |  |

{% hint style="info" %}
可預先設定「參數管理」的default\_inc\_group\_id，指定預設的受派組織ID
{% endhint %}

{% hint style="info" %}
系統內[事件管理](5.md#shi-jian-guan-li)可以設定觸發條件，將事件升級為事故並自動開單。
{% endhint %}
